README - Python Data Structures: Lists, Tuples, Sets & Dictionaries
======================================================================

INTRODUCTION & CONTEXT
----------------------
En Python, la plupart des programmes réels manipulent des collections
de valeurs plutôt que des variables isolées. Un programme capable de
stocker, grouper, transformer et interroger des données est plus
expressif, plus puissant et plus réutilisable.

Ce projet se concentre sur les structures de données intégrées les plus
importantes, ainsi que sur la logique nécessaire pour les choisir et les
utiliser correctement.

STRUCTURES ÉTUDIÉES
-------------------

Listes (list)
- Séquences ordonnées et mutables.
- Idéales pour stocker des éléments modifiables.
- Supportent l’indexation, le slicing et de nombreuses méthodes utiles.

Tuples (tuple)
- Séquences ordonnées mais immuables.
- Parfaits pour représenter des groupes de valeurs fixes.
- Utilisés pour garantir qu’un ensemble de données ne change pas.

Ensembles (set)
- Collections non ordonnées d’éléments uniques.
- Très efficaces pour tester l’appartenance ou éliminer les doublons.
- Permettent des opérations ensemblistes : union, intersection, etc.

Dictionnaires (dict)
- Mappages clé → valeur.
- Structure la plus flexible pour représenter des données associatives.
- Accès très rapide aux valeurs via les clés.

OBJECTIFS DU PROJET
-------------------

À la fin de ce projet, vous serez capable de :

1. Manipuler des listes : ajout, suppression, parcours, copie, slicing.
2. Utiliser des tuples pour regrouper des données immuables.
3. Exploiter les ensembles pour gérer des éléments uniques.
4. Construire et manipuler des dictionnaires pour organiser des données.
5. Choisir la structure adaptée selon le problème.
6. Écrire des fonctions prévisibles, sûres et faciles à tester.
7. Comprendre les implications mutabilité/immutabilité.
8. Éviter les pièges courants : aliasing, modification inattendue, etc.

PHILOSOPHIE DU PROJET
---------------------

Vous implémenterez de petites fonctions ciblées, chacune conçue pour
manipuler une structure de données de manière claire et déterministe.
L’objectif est de développer une intuition solide sur :

- Quand utiliser une liste plutôt qu’un tuple.
- Quand un set est plus efficace qu’une liste.
- Quand un dictionnaire est la structure la plus expressive.
- Comment écrire du code qui reste lisible et robuste.

CONCLUSION
----------

Ce projet constitue une base essentielle pour tous les programmes Python
qui manipulent des données. Maîtriser ces structures vous permettra
d’écrire du code plus propre, plus rapide et plus professionnel.
